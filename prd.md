# PRD — Lensora MVP

## Product Overview

### Product Name

Lensora

### Product Category

Photography Marketplace Platform

### Platform

Responsive Web Application

### Product Vision

Menjadi platform utama untuk menemukan photographer berdasarkan style, budget, dan lokasi dengan pengalaman yang lebih mudah, terpercaya, dan transparan dibanding Instagram.

Dalam jangka panjang, Lensora berkembang menjadi Operating System untuk industri photography yang membantu photographer mendapatkan client dan mengelola bisnis mereka secara lebih profesional.

---

# Problem Statement

## User Problems

Saat ini mayoritas pengguna mencari photographer melalui Instagram.

Tantangan yang sering muncul:

* Sulit menemukan photographer sesuai style yang diinginkan.
* Sulit membandingkan harga antar vendor.
* Informasi harga sering tidak transparan.
* Portfolio tersebar dan tidak terstruktur.
* Sulit mengetahui kualitas vendor sebelum transaksi.
* Komunikasi berulang melalui DM atau WhatsApp.

---

## Vendor Problems

Sebagian besar photographer masih mengelola bisnis secara manual.

Permasalahan yang umum:

* Bergantung pada Instagram untuk mendapatkan client.
* Booking dilakukan melalui WhatsApp.
* Jadwal booking tidak terorganisir.
* Banyak pertanyaan yang berulang.
* Sulit menampilkan portfolio secara profesional.
* Sulit mendapatkan exposure kepada calon client baru.

---

# Monetization Strategy

## Phase 1

### Transaction Fee

Platform mengambil komisi sebesar 5% dari setiap transaksi yang berhasil.

Tujuan:

* Mengurangi hambatan vendor untuk bergabung.
* Mempercepat akuisisi vendor.

---

# MVP Scope

## User Features

### Vendor Discovery

User dapat mencari photographer berdasarkan:

* Style
* Harga
* Lokasi
* Kategori
* Availability

---

### Vendor Detail Page

Menampilkan:

* Portfolio
* Harga
* Paket
* Rating
* Review
* Lokasi
* Style Tags

---

### Compare Vendor

User dapat membandingkan maksimal 3 vendor.

Data yang dibandingkan:

* Harga
* Rating
* Paket
* Style
* Delivery Time

---

### Style Matching

User mengunggah gambar referensi.

Sistem akan merekomendasikan vendor berdasarkan:

* Style tags
* Portfolio tags
* Kategori fotografi

MVP menggunakan pendekatan semi-manual, bukan AI kompleks.

---

### Booking

User dapat:

* Memilih paket
* Menentukan tanggal
* Menentukan lokasi
* Menambahkan catatan

---

### DP Payment

User membayar DP sebagai konfirmasi booking.

---

### Review & Rating

User dapat memberikan review setelah order selesai.

---

# Vendor Features

## Vendor Registration

---

## Profile Management

Vendor dapat mengelola:

* Nama
* Bio
* Lokasi
* Kontak
* Kategori

---

## Portfolio Management

Vendor dapat:

* Menambahkan style tags

* Reject booking
